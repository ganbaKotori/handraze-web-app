Setup the handraze-web-app repository on GitBash

The handraze-web-app repository contains the branches MASTER, DEV, DOCUMENTS, GH-PAGES, and PROTOTYPE.

To get started, open a folder where you want to clone the repository. I have a folder named github in my documents folder that contains all of my repositories. Navigate to the destination of the folder on gitBash and then enter the command:

git clone https://github.com/ganbaKotori/handraze-web-app.git

To switch to the DEV branch enter the command:

git fetch

Followed by

git checkout dev

This applies to the branches in the repo. When you are pushing your work, MAKE SURE YOU ARE THE CORRECT BRANCH!

Working with Branches

When working on a new feature for Handraze, you need to create a new branch with the dev branch as the parent.
To accomplish this, make sure you "git checkout" the dev branch.
Follow this with the command:

git checkout -b feature_branch

feature_branch can be any name related to the feature you are developing. This new branch is based off the dev branch. This means that all the files from the dev branch are in this new branch.
Once you're done with development in the feature branch, commit and push the branch to github.

git add .

git commit -m "enter your message here(don't remove these quotation marks)"

git push

Do these 3 git commands above within your feature branch and it should show up on gitHub
